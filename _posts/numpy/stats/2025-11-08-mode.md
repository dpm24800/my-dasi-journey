## ðŸ§® Mathematical Concept of Mode

### 1. Definition

The **mode** is the **value that appears most frequently** in a dataset.
In other words, it represents the **most common observation** â€” the value that occurs with the highest frequency.

Unlike mean and median, which require numerical order, the mode is meaningful for **both numerical and categorical data**.

---

### 2. Formula and Representation

There is **no explicit formula** for mode like mean or median; itâ€™s determined by counting frequencies.

Let ( f(x_i) ) denote the **frequency** (number of occurrences) of each observation ( x_i ).

[
\text{Mode} = x_i \text{ such that } f(x_i) \text{ is maximum}
]

---

### 3. Types of Datasets Based on Mode

| Type           | Description         | Example                        |
| -------------- | ------------------- | ------------------------------ |
| **Unimodal**   | One mode            | [2, 3, 3, 4, 5] â†’ Mode = 3     |
| **Bimodal**    | Two modes           | [1, 1, 2, 3, 3] â†’ Modes = 1, 3 |
| **Multimodal** | More than two modes | [1, 1, 2, 2, 3, 3]             |
| **No mode**    | All values unique   | [1, 2, 3, 4, 5]                |

---

### 4. Example 1 â€” Simple Dataset

Dataset: [4, 2, 4, 3, 2, 4, 1]

Frequencies:

| Value | Count |
| ----- | ----- |
| 1     | 1     |
| 2     | 2     |
| 3     | 1     |
| 4     | 3     |

âœ… **Mode = 4**, because it appears most frequently.

---

### 5. Example 2 â€” Bimodal Dataset

Dataset: [2, 3, 3, 4, 4, 5]

Frequencies:

| Value | Count |
| ----- | ----- |
| 2     | 1     |
| 3     | 2     |
| 4     | 2     |
| 5     | 1     |

âœ… **Modes = 3 and 4** (both appear twice).

---

### 6. Usefulness

* For **categorical data** (like colors, cities, brands, etc.) where mean/median make no sense.
* For identifying **most common choices or behaviors**.
* To describe **nominal-level data** (non-numeric labels).

---

### 7. Relation to Distribution Shape

| Distribution Type     | Relationship         |
| --------------------- | -------------------- |
| **Symmetrical**       | Mean = Median = Mode |
| **Positively Skewed** | Mode < Median < Mean |
| **Negatively Skewed** | Mean < Median < Mode |

Thus, mode helps describe **distribution skewness** along with mean and median.

---

### 8. Limitation

* Mode might **not be unique** (multiple modes).
* It can be **unstable** if small changes in data alter frequency counts.
* For continuous data, mode is found by **grouping or histogram estimation**.

## ðŸ’» NumPy / SciPy Implementation

NumPy does **not** include a dedicated mode function.
Instead, we use `scipy.stats.mode()` (from the SciPy library), which provides a robust and flexible implementation.

---

### 1. Function: `scipy.stats.mode()`

**Syntax:**

```python
scipy.stats.mode(a, axis=0, nan_policy='propagate', keepdims=False)
```

### 2. Parameters

| Parameter      | Description                                               |
| -------------- | --------------------------------------------------------- |
| **a**          | Input array or sequence                                   |
| **axis**       | Axis along which to compute mode                          |
| **nan_policy** | How to handle NaN values: â€˜propagateâ€™, â€˜omitâ€™, or â€˜raiseâ€™ |
| **keepdims**   | Keep reduced dimensions for broadcasting                  |

---

### 3. Example 1: Basic Mode

```python
import numpy as np
from scipy import stats

data = np.array([4, 2, 4, 3, 2, 4, 1])
mode_result = stats.mode(data, keepdims=True)

print("Mode:", mode_result.mode[0])
print("Frequency:", mode_result.count[0])
```

**Output:**

```
Mode: 4
Frequency: 3
```

âœ… The most frequent value is **4**, appearing **3 times**.

---

### 4. Example 2: Handling Multidimensional Data

```python
matrix = np.array([[1, 2, 2],
                   [3, 3, 4],
                   [5, 5, 5]])

# Compute mode along columns
mode_col = stats.mode(matrix, axis=0, keepdims=True)
print("Column-wise mode:", mode_col.mode)
```

**Output:**

```

```

---

### 5. Example 3: Mode with Missing Values (NaN Handling)

```python
data = np.array([1, 2, np.nan, 2, 3, 2, np.nan])
mode_result = stats.mode(data, nan_policy='omit', keepdims=True)

print("Mode (ignoring NaN):", mode_result.mode[0])
```

**Output:**

```
Mode (ignoring NaN): 2.0
```

âœ… `nan_policy='omit'` ensures missing data does not affect the result.

---

### 6. Example 4: Custom Frequency Check (Manual Count)

You can manually compute the mode using `numpy.unique()` if SciPy isnâ€™t available:

```python

```

**Output:**
